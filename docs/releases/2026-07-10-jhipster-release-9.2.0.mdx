# JHipster release v9.2.0

This is a minor release for JHipster v9.

It includes [503 closed issues and pull requests](https://github.com/jhipster/generator-jhipster/issues?q=is:closed+milestone:9.2.0) after the [9.1.0 release](/2026/05/27/jhipster-release-9.1.0.html).

## What's new?

* Our 3000+ EJS templates are now [checked and formatted by eslint](https://github.com/jhipster/generator-jhipster/issues/32851)
* Upgraded to [Spring Boot 4.0.7](https://github.com/jhipster/generator-jhipster/pull/33753) and improved generated backend hardening with a [safer AccountResource](https://github.com/jhipster/generator-jhipster/pull/33671)
* Made Vitest the only frontend test framework for generated apps with [Jest removal from client choices](https://github.com/jhipster/generator-jhipster/pull/34017) and [Angular cleanup of leftover Jest conditionals](https://github.com/jhipster/generator-jhipster/pull/34006)
* Included [an application generation fix](https://github.com/jhipster/generator-jhipster/pull/33507) for initial project generation
* Fixed generated frontend behavior across Angular, React, and Vue with [PUT support for entities without autogenerated IDs](https://github.com/jhipster/generator-jhipster/pull/34013), a [React entity creation fix](https://github.com/jhipster/generator-jhipster/pull/34029), and a [Vue Boolean field rendering fix](https://github.com/jhipster/generator-jhipster/pull/33759)
* Improved blueprint generation and internals with [stronger generate-blueprint typing and compatibility](https://github.com/jhipster/generator-jhipster/pull/33927), [reworked bootstrap generator composition](https://github.com/jhipster/generator-jhipster/pull/33688), and [MemFsEditorFile integration](https://github.com/jhipster/generator-jhipster/pull/33922)
* Upgraded build tooling to [Node 24.18.0](https://github.com/jhipster/generator-jhipster/pull/33880), [Gradle 9.6.1](https://github.com/jhipster/generator-jhipster/pull/33915), and [webpack-dev-server 6](https://github.com/jhipster/generator-jhipster/pull/34007)

## How to install

To install JHipster v9.2.0 using NPM:

```
npm install -g generator-jhipster@9.2.0
```

It is also available using the JHipster Docker image, as it is automatically built from our source code.

- [Docker Hub](https://hub.docker.com/r/jhipster/jhipster)
- [JHipster Online](https://start.jhipster.tech)

## How to upgrade

**Automatic upgrade**

For an automatic upgrade, use the [JHipster upgrade sub-generator](/upgrading-an-application/) on an existing application:

Upgrade your version of JHipster:

```
npm update -g generator-jhipster
```

And then run the upgrade sub-generator:

```
jhipster upgrade
```

You can also use the [migrate blueprint](https://github.com/jhipster/generator-jhipster-migrate) for more advanced upgrade features.

```
npm i -g generator-jhipster-migrate
jhipster-migrate
```

**Manual upgrades**

For a manual upgrade, first upgrade your version of JHipster with:

```
npm update -g generator-jhipster
```

If you have an existing project, it will still use the JHipster version with which it was generated.
To upgrade your project, you must first delete its `node_modules` folder and then run:

```
jhipster
```

You can also update your project and all its entities by running:

```
jhipster --with-entities
```

You can also update your entities one-by-one by running again the entity sub-generator, for example if your entity is named _Foo_:

```
jhipster entity Foo --single-entity
```
