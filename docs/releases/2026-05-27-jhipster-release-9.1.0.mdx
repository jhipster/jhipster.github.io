# JHipster release v9.1.0

This is a minor release for JHipster v9.

It includes [869 closed issues and pull requests](https://github.com/jhipster/generator-jhipster/issues?q=is:closed+milestone:9.1.0) after the [9.0.0 release](/2026/03/10/jhipster-release-9.0.0.html).

## What's new?

* Upgrade to Spring Boot [4.0.5](https://github.com/jhipster/generator-jhipster/pull/33159)
* Harden generated applications with [configuration sanitization for security](https://github.com/jhipster/generator-jhipster/pull/32708)
* Improve generated Angular apps with [OnPush change detection in all components](https://github.com/jhipster/generator-jhipster/pull/33218) and a [rewritten service worker provider](https://github.com/jhipster/generator-jhipster/pull/33329)
* Migrate React apps from [Jest to Vitest](https://github.com/jhipster/generator-jhipster/pull/33200)
* Continue Vue microfrontend modernization with [@module-federation/vite](https://github.com/jhipster/generator-jhipster/pull/33283), [rsbuild microfrontends by default](https://github.com/jhipster/generator-jhipster/pull/33455), and [gateway resource loading through module federation](https://github.com/jhipster/generator-jhipster/pull/33485)
* Upgrade build tools to [Node 24.16.0](https://github.com/jhipster/generator-jhipster/pull/33505), [Gradle 9.5.1](https://github.com/jhipster/generator-jhipster/pull/33382), and [Maven 3.9.16](https://github.com/jhipster/generator-jhipster/pull/33439)

## How to install

To install JHipster v9.1.0 using NPM:

```
npm install -g generator-jhipster@9.1.0
```

It is also available using the JHipster Docker image, as it is automatically built from our source code.

- [Docker Hub](https://hub.docker.com/r/jhipster/jhipster)
- [JHipster Online](https://start.jhipster.tech)

## How to upgrade

**Automatic upgrade**

For an automatic upgrade, use the [JHipster upgrade sub-generator](/upgrading-an-application/) on an existing application:

Upgrade your version of JHipster:

```
npm update -g generator-jhipster
```

And then run the upgrade sub-generator:

```
jhipster upgrade
```

You can also use the [migrate blueprint](https://github.com/jhipster/generator-jhipster-migrate) for more advanced upgrade features.

```
npm i -g generator-jhipster-migrate
jhipster-migrate
```

**Manual upgrades**

For a manual upgrade, first upgrade your version of JHipster with:

```
npm update -g generator-jhipster
```

If you have an existing project, it will still use the JHipster version with which it was generated.
To upgrade your project, you must first delete its `node_modules` folder and then run:

```
jhipster
```

You can also update your project and all its entities by running:

```
jhipster --with-entities
```

You can also update your entities one-by-one by running again the entity sub-generator, for example if your entity is named _Foo_:

```
jhipster entity Foo --single-entity
```
